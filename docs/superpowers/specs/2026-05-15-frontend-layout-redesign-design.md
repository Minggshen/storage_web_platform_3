# 前端六页面布局优化 · 设计规格

## 概述

对 6 个流程页面（项目总览、拓扑建模、资产绑定、构建校验、计算运行、结果展示）进行布局优化，使卡片排布更美观整洁，每页划分为清晰的逻辑步骤，引导用户逐步建立仿真工程。

## 全局规则

### 页面宽度

所有页面内容区统一使用全宽（与 AppShell 主内容区一致），`max-width` 与拓扑建模页当前宽度对齐。

### 卡片样式

- Tailwind 页面（ProjectOverviewPage, AssetsPage, BuildPage, SolverPage）：使用 `rounded-2xl border border-border bg-card p-5`
- inline style 页面（TopologyPage, ResultsPage）：保持现有 inline style 风格，仅调整布局结构
- 卡片间距统一为 `gap-5`（Tailwind）或 `16px`（inline style）

### 步骤编号

所有页面使用统一的步骤编号圆圈（①②③④），通过新增的 `StepBadge` 组件渲染：

```
frontend/src/components/common/StepBadge.tsx
```

组件接口：
```tsx
<StepBadge step={1} label="步骤标题" />
```

渲染为蓝色圆圈编号 + 标题文字的 flex 行，字号和圆圈大小可跟随上下文缩放。

### 侧边栏导航

**保持不动**。AppShell 的 6 步导航（`navItems` + `Progress` 进度条）已在之前的实现中完成，包括 10 秒轮询 dashboard 保持步骤状态同步。本次仅改内容区页面，不碰 AppShell。
