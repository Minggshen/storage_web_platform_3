# 创新点清单：配电网节点差异化负荷建模与典型日模型库构建方法

## 项目概述

本项目提出了一套面向配电网储能优化配置的节点级负荷建模方法，针对工商业用户和居民用户用电行为驱动机制的差异，分别设计了差异化的建模路径。工商业路径以"公司主工休模式"为组织核心，通过周内工休聚类→公司主模式推断→标签修正→年度峰谷分段的层次化时序分解，最终以"工休类型×年负荷段"的组合交叉方式构建典型日模型库。居民路径则通过多维特征工程（24维形状特征+16维统计/时段/时序特征）配合自适应 KMeans 聚类与语义化模型命名，生成可直接供求解器使用的全年逐日模型映射表与模型曲线库。两条路径统一输出为求解器可消费的 runtime 格式（year_model_map + model_library），实现了从原始电表数据到优化模型输入的全自动化。

## 候选创新点及调研验证

### 创新点 1：工商业用户"周内聚类→主模式推断→多级修正"的层次化工休模式识别方法

**技术问题**：工商业用户的用电行为高度依赖于其生产/经营作息（如五天工作制、轮班制、连续生产等），但实际负荷数据中常混杂异常周（节假日调休、临时停工、设备检修等），使得直接基于单一周内聚类的结果不可靠。现有方法通常仅做简单的工作日/非工作日二分类，未考虑"公司层面的主导工休模式"如何用于纠正个别异常周的误判。

**本项目的技术方案**：

提出三阶段层次化工休模式识别方法：

**阶段一——逐周独立聚类**：对每周 7 天进行 KMeans（k=2）聚类，使用由 24 维形状特征（归一化小时曲线）和 5 维统计特征（日均值/峰值/谷值/标准差/负荷率）拼接而成的特征向量。采用三重阈值联合判别（轮廓系数≥0.12 且（曲线差异≥0.05 或能量差异≥0.04））决定是否采纳二分类——阈值不满足则整周判为"同类日"（工休不分），满足则通过周末占比自动标定"工作日"和"休息日"。

**阶段二——公司主工休模式推断**：将所有完整周的逐日标签编码为 7 字符模式串（如"WWWWWRR"表示周一至周五工作日、周末休息日），统计全年各模式的出现频次，以多数投票确定公司主工休模式。

**阶段三——三级标签修正**：对每周的初判标签进行重审：(a) 与主模式完全一致的周→保留初判，标记"主模式周"；(b) 与主模式不一致但轮廓系数≥0.15 的周→保留初判，标记"特殊周"（可能为真实的生产异常）；(c) 偏离主模式且聚类置信度不足的周→按主模式强制修正，标记"主模式修正周"。

**相关现有技术调研**：

- 专利 CN202410104843（国网金华，2024）："基于用户用电行为分析的有序用电决策方法"——基于工作日/周休日分别进行 K 均值聚类，但仅做简单的二分类，未涉及公司主导模式推断或异常周修正。
- 专利 CN118626883A（2024）："负荷数据的特性分析方法"——对工作日和节假日分别序列聚类后加权合成，同样为简单二分类，无模式推断和修正。
- 专利 US20030101009（Johnson Controls，2003）：通过 GESD 异常检测 + 凝聚层次聚类自动发现每周各天的相似性分组，可识别出"周一至周五一组、周末一组"等模式。与本方案最接近，但其方法：(a) 无"公司主模式"概念——不会用多数模式去修正异常周；(b) 依赖层次聚类合并而非周级聚类+模式串编码；(c) 未区分"特殊周"和"可修正周"。
- "基于工商业用户负荷的生产行为分群分类方法"专利：建立了非正常工作日识别模型和白班/夜班/连续生产分类，侧重生产班次类型，非工休模式识别。

**新颖性分析**：

现有方法的核心逻辑是"对每日/每周独立聚类"，而本方案的核心逻辑是"聚类→全局投票→回写修正"。这种"从局部到全局再回到局部"的层次化纠偏策略在现有专利中未见。特别是：(a) 7 字符模式串编码使周模式可直接做字符串比对和频次统计；(b) 三级修正规则中"特殊周"与"可修正周"的区分——以轮廓系数 0.15 为界——为处理真实生产异常（如赶工、检修停工）与聚类噪声提供了差异化的处理策略。

**项目中的体现位置**：
- `backend/services/load_modeling_industrial.py:188-273`：`classify_one_week()` 周内工休聚类
- `backend/services/load_modeling_industrial.py:299-344`：`infer_company_main_workrest_pattern()` 公司主模式推断
- `backend/services/load_modeling_industrial.py:347-401`：`revise_week_labels_by_main_pattern()` 三级标签修正

### 创新点 2：基于"工休类型×年负荷段"组合交叉的工商业典型日模型库构建方法

**技术问题**：工商业用户全年负荷受两个独立维度的影响——周内工休节奏（决定短期用电模式）和年度季节性负荷水平变化（决定长期用电量级）。现有方法通常将两者混为一维聚类，导致模型库中同时出现"夏季工作日"和"冬季休息日"等语义混杂的模型，难以解释且不利于下游储能优化调度。

**本项目的技术方案**：

提出两阶段独立分解后组合交叉的方法：

**维度一——周内工休类型**：经创新点 1 的层次化修正后，全年每一天被标注为"工作日""休息日"或"同类日"之一（3 类）；

**维度二——年度峰谷分段**：以周为粒度，提取每周的 6 维负荷特征（周平均/峰值/谷值/标准差/日峰谷差/日均负荷），使用 KMeans（k=2-5，轮廓系数自动择优，阈值 0.18）将全年 52 周划分为高负荷期、低负荷期及若干中间期（1-5 类）；

**组合交叉**：将两个维度的标注在日粒度上做笛卡尔积——每一天得到一个形如"(工休编码, 年类编码)"的组合模型编号（如"(1,1)"="工作日-高负荷期"）。对每种组合下所有天的 24 小时曲线取均值，构建"组合典型日负荷模型库"。

该方法的核心优势在于：(a) 两个维度各自独立建模，互不干扰；(b) 组合模型具有清晰的语义解释（某种作息类型 + 某个负荷水平段）；(c) 最终模型数量 = 工休类型数 × 年类数，而非不可控的聚类结果。

**相关现有技术调研**：

- 专利 CN118626883A（2024）：对工作日和节假日分别聚类，但未进行年度季节性维度的独立分解，也未做维度交叉组合。
- 专利"一种面向储能规划的典型复合型场景缩减方法"：分别对风电、光伏、负荷做 KMeans 聚类后重新组合为复合典型场景——思路最接近，但应用对象是风电/光伏/负荷的跨能源类型组合，而非同一用户负荷的两个时间维度交叉。且其使用概率加权组合，非笛卡尔积。
- "基于多维 Isolate-Detect 多变点检测的企业用电典型模式分析方法"：提取工作日和周末差异，但仅做变点检测分段，不涉及双维度独立建模和交叉组合。

**新颖性分析**：

"两个维度独立分解后组合交叉"的方法论在聚类领域属于降维/解耦的经典思路，但在工商业用户负荷建模这一特定应用中，将其应用于"工休类型×年负荷段"构成语义清晰的组合模型库，在现有专利检索中未见完全相同的方案。关键差异在于现有方法要么将不同来源的数据（风/光/荷）各自聚类后组合，要么在单一维度上做工作日/休息日区分——均未实现同一用户负荷的"日内行为模式"与"年度量级变化"两个时间维度的独立建模与笛卡尔积组合。

**项目中的体现位置**：
- `backend/services/load_modeling_industrial.py:426-481`：`classify_annual_periods()` 年度峰谷分段
- `backend/services/load_modeling_industrial.py:484-515`：`build_final_daily_mapping()` 双维度组合映射
- `backend/services/load_modeling_industrial.py:518-547`：`build_model_library()` 组合模型库构建

### 创新点 3：工商业-居民差异化双路径负荷建模框架

**技术问题**：配电网中同时存在工商业节点和居民节点，两者的用电行为驱动机制根本不同——工商业受生产/经营作息驱动（规律性强、工休差异显著），居民受生活作息和季节气候驱动（个体差异大、无明显工休边界）。使用统一的建模方法（如直接对全年做 KMeans）无法准确捕捉各自的行为特征。

**本项目的技术方案**：

提出按节点类型分流的两条差异化建模路径：

**工商业路径**（层次化时序分解）：
- 特征：24 维形状特征 + 5 维统计特征
- 方法：周内聚类→主模式推断→标签修正→年度分段→双维度交叉组合
- 模型粒度：工休类型(≤3) × 年负荷段(≤5) = ≤15 个组合模型
- 输出语义：如"工作日-高负荷期""休息日-低负荷期"

**居民路径**（多维特征聚合聚类）：
- 特征：24 维形状特征 + 16 维扩展特征（统计特征 5 维 + 五时段占比 5 维 + 周末/节假日旗标 2 维 + 月份 sin/cos 编码 2 维）
- 方法：自适应 KMeans（k=2-8，轮廓系数择优，阈值 0.08 回退为 1 类）
- 模型粒度：典型 ≤8 个居民日类型
- 输出语义：自然语言模型命名（季节属性 + 日期属性 + 曲线形状属性，如"夏季工作日型晚高峰型"）

两条路径最终统一输出为 solver runtime 格式（`runtime_year_model_map.csv` + `runtime_model_library.csv`），确保下游求解器可无差别消费。

**相关现有技术调研**：

- 现有负荷建模专利几乎全部面向单一用户类型（全为工商用户/全为居民用户/全为区域级聚合负荷），未见在同一框架内按节点类型分流并采用差异化建模策略的方案。
- 专利 CN119577634A（华南理工，2025）：构建行业多时间尺度典型用电曲线，涉及行业标签辅助聚类，但仍为统一方法框架，非按类型分流的双路径策略。
- 学术文献中的差异化负荷建模通常针对"商业建筑 vs 住宅建筑"的能耗预测，使用不同的预测模型（LSTM vs CNN 等），但非面向储能优化的典型日模型库构建。

**新颖性分析**：

"分流+差异化建模+统一输出"的框架在以下方面具有新颖性：(a) 分流依据是节点用电行为的驱动机制（作息驱动 vs 生活驱动），而非简单的行业分类；(b) 两条路径在特征工程、聚类策略、模型粒度和语义命名上完全独立设计，而非统一方法套不同参数；(c) 统一输出格式（runtime CSV）使差异化建模结果对下游求解器透明——这是面向储能优化这一特定下游任务的工程设计，现有专利未涉及。需注意，"分流建模"作为框架思路的专利保护范围可能较窄，建议与创新点 1、2 合并，以工商业路径的层次化方法为核心创新点。

**项目中的体现位置**：
- `backend/services/load_data_processing_service.py:51-150`：`process_all_nodes()` 按节点类别分流
- `backend/services/load_modeling_industrial.py:682-776`：工商业路径 `process_one_company()`
- `backend/services/load_modeling_residential.py:548-629`：居民路径 `process_one_company()`
- `backend/services/build_runtime_industrial.py` + `build_runtime_residential.py`：统一 runtime 转换

### 创新点 4：（备选）面向储能调度优化的负荷模型 runtime 编码与消费格式

**技术问题**：典型日负荷建模的输出格式各异（Excel 多 sheet、PNG 图表等），无法直接被优化求解器消费。需要一种紧凑、确定性的编码格式，使建模结果可以无缝接入年度 8760 小时储能调度仿真。

**本项目的技术方案**：

定义统一的 runtime 双文件格式：(1) `runtime_year_model_map.csv`：365 行，每行为 `date, day_index, internal_model_id, external_model_id, model_name`，将全年每一天映射到唯一整数模型 ID；(2) `runtime_model_library.csv`：N 行，每行为 `internal_model_id, external_model_id, model_name, h00, h01, ..., h23`，存储每个模型的 24 小时负荷值。求解器通过 `day_index` 查找模型 ID，再从模型库读取 24 点曲线，实现 O(1) 的逐日负荷查询。

此格式的关键设计：(a) `internal_model_id` 为连续整数（0, 1, 2, ...），使求解器可直接用数组索引而非字典查找；(b) `external_model_id` 保留原始建模标签（如"(1,1)"或"R03"），保持可追溯性；(c) 双文件分离使模型库可跨场景复用。

**新颖性分析**：

此创新点属于数据格式/接口层面的工程优化，方法层面的创新性较弱。建议作为创新点 1-3 的输出格式说明出现，而非独立创新点。

## 建议

**首选方案：创新点 1 + 创新点 2 合并为一个专利**，覆盖"工商业用户层次化工休模式识别与双维度组合典型日模型库构建方法"的完整技术路线。这两个创新点在技术流程上前后衔接（层次化工休识别 → 年度分段 → 交叉组合），合并后形成完整的工商业负荷建模闭环。

创新点 3（双路径框架）可作为专利的扩展实施例，说明该方法在工商业和居民两类节点上的差异化适配。
