# 专利技术交底书

## 一、专利名称

一种基于层次化工休模式识别与双维度组合的工商业用户典型日负荷模型库构建方法

## 二、发明人/申请人/地址

（留空）

## 三、本专利的应用领域

本发明涉及配电网负荷建模与储能优化配置领域，尤其涉及一种面向工商业用户的基于逐周工休聚类、公司主模式推断与标签修正的层次化工休模式识别方法，以及基于"工休类型×年负荷段"双维度独立分解后组合交叉的典型日负荷模型库构建方法，可用于配电网节点级储能容量优化配置的全生命周期仿真评估。

## 四、本专利的任务/要解决的技术问题

在配电网储能优化配置中，需要对每个接入节点的全年负荷行为进行建模，以便在年度调度仿真中为储能容量优化提供尽可能准确且可解释的负荷输入。工商业用户——包括工厂、写字楼、商场、仓储物流等——是配电网中负荷密度最高的节点类型，其用电行为高度依赖于生产/经营作息制度和季节性负荷水平变化。准确提取这些用户的用电行为规律并构建紧凑的典型日负荷模型库，直接决定了储能容量配置结果的可靠性。

然而，工商业用户的实际负荷数据面临三个核心困难。第一，周内工休节奏与年度季节性变化是两个相互独立的时间维度，但现有方法通常将两者混为一维聚类，导致模型语义混杂——例如"夏季工作日"和"冬季休息日"可能因总用电量相近被归入同一聚类，但其负荷曲线形态和峰谷时段截然不同，这对储能充放电策略的制定是致命的。第二，实际负荷数据中包含大量异常周——节假日调休导致的工作日/休息日互换、临时停工检修、高温限电等——单周独立聚类无法区分"真实的生产异常"和"聚类统计噪声"，容易产生误判。第三，储能优化面向的是10至15年的全生命周期，如果直接使用某单一年份的原始负荷数据驱动全生命周期仿真，相当于将单年的偶然性——极端天气、临时停产、设备故障——外推到整个项目周期，导致配置结果对不同年景数据高度敏感（鲁棒性差）且无法追溯决策依据（可解释性差）。

本专利的任务是：（1）提出一种层次化工休模式识别方法，从逐周独立聚类出发，通过全局模式编码与多数投票提取公司层面的主导工休模式，再利用三级修正规则回写纠偏各周标签，解决单周聚类的局部误判问题；（2）提出一种双维度独立分解后组合交叉的模型库构建方法，将工休维度与年度峰谷维度分别建模后做笛卡尔积，构建语义清晰的组合典型日模型库，解决一维聚类导致的模型语义混杂问题；（3）通过提取用电行为的结构性规律而非简单使用原始数据，使负荷模型具有跨年鲁棒性和储能调度决策的可解释性，支撑可靠的全生命周期储能优化配置。

## 五、已有技术/产品的不足

在现有负荷建模与分类技术中，基于K均值聚类的典型日提取方法已被广泛采用。如专利CN202410104843（国网浙江省电力有限公司金华供电公司，2024）提出基于典型负荷形态编码构建用户负荷字典，分别对工作日和周休日进行K均值聚类以提取典型用电行为，但该方法仅做简单的工休二分类判别，未考虑不同周之间的标签一致性，也未引入公司层面的全局行为模式来纠正个别异常周的聚类误判。专利CN118626883A（2024）公开了一种负荷数据特性分析方法，对工作日负荷数据和节假日负荷数据分别进行序列聚类后加权合成典型最优负荷特性曲线，但该方法同样停留在单一维度的二分类层面，未能将周内工休行为与年度季节性变化这两个正交时间维度解耦处理，且节假日判定依赖固定日历规则，无法自适应识别不同企业的个性化作息制度。在工休模式的自适应识别方面，美国专利US20030101009（Johnson Controls Technology Company，2003）提出了一种基于GESD异常检测和凝聚层次聚类的日类型自动分组方法，通过去除季节性趋势后按星期几进行初始分组再逐步合并相似组，能自动发现"周一至周五一组、周末一组"等模式，其不足之处在于仅依赖层次聚类的合并/停止规则，缺乏"多数投票确定全局主模式、再回写修正局部标签"的纠偏机制，且未考虑年度季节性分段与工休维度的组合建模。专利CN119577634A（华南理工大学，2025）提出了基于立体画像库和双通道加权的用户异常负荷辨识方法，通过聚类提取用户多时间尺度典型用电曲线并结合行业标签二次聚类，但其侧重点在于负荷异常检测而非典型日模型库的构建，也未涉及双维度解耦和组合交叉的建模思路。基于多维Isolate-Detect多变点检测的企业用电典型模式分析方法（福建师范大学，2024）采用变点检测技术划分用电时段并提取工作日与周末的典型用电行为差异，但该方法依赖单日数据的统计变点，对数据完整性要求高，且对异常周的识别和修正能力有限。此外，上述方法均面向负荷分析或异常检测等任务，未考虑下游储能全生命周期优化对负荷模型的特殊需求——即负荷模型需具备跨年鲁棒性和决策可解释性，而直接使用原始数据进行聚类或预测均无法满足这一要求。因此，有必要提出一种专门面向储能优化配置场景的、能够自适应识别企业工休模式并进行全局纠偏、同时将工休维度和季节维度解耦后组合构建语义清晰模型库的负荷建模方法。

## 六、本专利的内容

### 6.1 总体方案

本发明提出一种基于层次化工休模式识别与双维度组合的工商业用户典型日负荷模型库构建方法。如图1所示，该方法的整体技术路线为：首先对原始电表数据进行预处理，将原始采样频率统一重采样为1小时间隔的标准化负荷序列，并构建以日为单位、每日24个时点的日负荷矩阵；然后进入层次化工休模式识别阶段——先逐周独立聚类判断每周内是否存在显著的工休日类型差异，再将所有完整周的聚类结果编码为7字符模式串进行全局统计，以多数投票确定该公司的主工休模式，最后以主模式为基准对各周标签执行三级修正；在完成工休标注后，进入年度峰谷分段阶段——以周为粒度构建负荷特征，通过自适应聚类将全年各周划分为不同的季节性负荷水平段；随后进入双维度组合模型库构建阶段——将工休类型维度与年度峰谷类型维度做笛卡尔积交叉组合，对每种组合下的所有天取24小时均值，生成组合典型日模型库；最后将模型库及全年逐日模型映射表统一输出为标准化runtime格式，供下游储能优化求解器消费。以下结合附图和公式对各技术模块进行详细说明。

### 6.2 数据预处理与日负荷矩阵构建

对于输入的单个工商业用户全年原始电表数据（通常为Excel格式，第一列为时间戳、第二列为负荷有功功率值），首先进行数据清洗：解析时间列并排序，去除重复时间戳的冗余记录，检测并插值填充缺失时段。设原始采样频率为 f₀（通常为15分钟），目标建模粒度为 1 小时，则重采样后的全年数据为：

```
L = {l_{d,h} | d ∈ {1,2,...,365}, h ∈ {0,1,...,23}}    (1)
```

式中，l_{d,h} 为第 d 天第 h 小时的负荷功率（kW）。将全年数据组织为 365×24 的日负荷矩阵 M：

```
M[d, h] = l_{d,h}                                        (2)
```

式中 M 的每一行代表一天的 24 小时负荷曲线。同时从时间戳中提取每个日期 d 对应的星期序号 w(d) ∈ {0,1,...,6}（0=周一，6=周日）和月份 m(d) ∈ {1,2,...,12}。

### 6.3 层次化工休模式识别

本模块是本发明方法的核心创新之一，包含逐周独立聚类、公司主工休模式推断、三级标签修正三个递进阶段，如图2所示。

#### 6.3.1 逐周独立聚类

以自然周（周一至周日）为单位，将全年划分为若干周。对每一起始日为 w₀ 的周，提取该周内各天的 24 小时负荷曲线，构成日负荷子矩阵 M_week，其行数为该周的天数 n（通常 n=7，首尾周可能不足 7 天）。

对每一周，构建日级特征向量。对于矩阵 M_week 中的第 i 行（对应第 i 天），计算：

（a）形状特征：24维归一化小时曲线

```
f_shape(i, h) = M_week[i, h] / mean(M_week[i, :])       (3)
```

式中 mean(M_week[i,:]) 为该天 24 小时负荷均值。形状特征刻画了日负荷曲线的"形态"而不受绝对量级影响。

（b）统计特征：5维，包括

```
f_mean(i) = mean(M_week[i, :])                          (4)
f_peak(i) = max(M_week[i, :])                           (5)
f_valley(i) = min(M_week[i, :])                         (6)
f_std(i) = std(M_week[i, :])                            (7)
f_loadfactor(i) = f_mean(i) / max(f_peak(i), ε)          (8)
```

式中 ε 为极小正数防止除零，std(·) 为标准差，f_loadfactor 为日负荷率（均值与峰值之比）。

将 24 维形状特征与 5 维统计特征拼接为 29 维特征向量 F_i ∈ R²⁹。对特征矩阵 F ∈ R^{n×29} 进行标准化后，执行 KMeans 聚类（k=2）：

```
labels_2 = KMeans(k=2, random_state=42).fit_predict(StandardScaler(F))    (9)
```

为判断该周是否确实存在显著的工休二元差异，引入三重阈值联合判别。计算聚类轮廓系数 S、曲线差异指标 G_profile 和能量差异指标 G_energy：

```
S = silhouette_score(StandardScaler(F), labels_2)       (10)

G_profile = mean(|C₀ - C₁|) / mean_level                (11)

G_energy = |mean(M[C₀]) - mean(M[C₁])| / mean_level     (12)
```

式中 C₀ 和 C₁ 分别为两个簇的聚类中心 24 小时曲线向量，mean_level 为本周全部天数的负荷均值。

当且仅当同时满足以下条件时，认为该周存在明显的工休差异，采纳二分类结果：

```
S ≥ 0.12 且 (G_profile ≥ 0.05 或 G_energy ≥ 0.04)      (13)
```

若条件(13)不满足，则整周所有天均标注为"同类日"（类别编码 3），表示该周内各天用电行为无显著差异。

若条件(13)满足，则进一步确定工作日与休息日的对应关系。设簇 0 的周末（周六、周日）天数占比为 r₀，簇 1 的周末天数占比为 r₁。若 r₀ > r₁，则簇 0 为休息日、簇 1 为工作日；若 r₁ > r₀，则簇 1 为休息日、簇 0 为工作日；若两者相等，则取日均负荷均值较低的簇为休息日（休息日通常负荷更低）。工休标注结果记为：工作日（类别编码 1）、休息日（类别编码 2）。

#### 6.3.2 公司主工休模式推断

对所有完整周（包含全部 7 天、无缺失数据的周），将逐日工休标注结果编码为 7 字符模式串。编码规则为：

```
W = 工作日（类别编码 1）
R = 休息日（类别编码 2）
S = 同类日（类别编码 3）
M = 缺失日（数据不完整）
```

例如"WWWWWRR"表示周一至周五为工作日、周六周日为休息日的标准五天工作制模式；"WWWWWWR"表示周一至周六工作、周日休息的六天工作制；"SSSSSSS"表示全周无工休差异的连续生产模式。

统计全年所有完整周中各模式串的出现频次。设模式串集合为 {P₁, P₂, ..., P_K}，每个模式串的出现次数为 c(P_k)。以频次最高的模式串作为该公司的主工休模式 P*：

```
P* = argmax_{P_k} c(P_k)                                (14)
```

若出现频次并列，优先选择工作日天数更多的模式作为 P*（保守原则——将更多天视为工作日有利于储能容量的安全裕度）。

#### 6.3.3 三级标签修正

以确定的主工休模式 P* 为基准，对各周执行三级标签修正，如图3所示。

**第一级——主模式周**：若某周的初判模式串 P_week 与主模式 P* 完全一致（逐天标签均匹配），则该周的所有天保留初判标签，标记为"主模式周"。该周内各天的最终工休类别等于初判类别。

**第二级——特殊周**：若某周的初判模式串与主模式不一致，但该周逐周聚类的轮廓系数 S_week ≥ 0.15（特殊周阈值），则认为该周虽然偏离主模式，但聚类置信度较高，可能反映真实的生产行为异常（如赶工加班、设备检修停工等），保留初判标签，标记为"特殊周"。

**第三级——主模式修正周**：若某周的初判模式串与主模式不一致，且轮廓系数 S_week < 0.15，则判定该周的偏离系聚类统计噪声或低质量聚类所致，应将各天标签按主模式 P* 的对应位置强制改写。即对于该周的第 j 天（j=0,...,6），若 P*[j] = 'W'，则该天最终标注为"工作日"；若 P*[j] = 'R'，则标注为"休息日"；若 P*[j] = 'S'，则标注为"同类日"。标记为"主模式修正周"。

对于首尾不完整周（不足 7 天），直接按主模式 P* 对应位置赋值，不做独立聚类。

完成三级修正后，全年每一天均获得一个最终工休类别标签 c_workrest(d) ∈ {1(工作日), 2(休息日), 3(同类日)}。

### 6.4 年度周尺度峰谷分段

与工休维度的逐天标注并行，本模块以周为粒度对全年进行季节性负荷水平分段，如图4所示。

对每周提取 6 维周负荷特征向量：

```
f_weekly(week) = [
    mean(M_week),      // 周平均负荷
    max(M_week),       // 周峰值负荷
    min(M_week),       // 周谷值负荷
    std(M_week),       // 周负荷标准差
    mean(peak - valley), // 周平均日峰谷差
    mean(day_mean)       // 周日均负荷均值
]                                                        (15)
```

对全部 N_weeks 周的 6 维特征矩阵（大小为 N_weeks × 6）进行标准化处理后，执行 KMeans 聚类，在 k = 2 至 min(5, N_weeks-1) 的范围内，对每个 k 计算聚类轮廓系数 S_year(k)。选出使 S_year(k) 最大且聚类后各簇样本数均不小于 2 的 k* 作为最优聚类数：

```
k* = argmax_{k ∈ [2, min(5, N_weeks-1)]} S_year(k)       (16)
```

若最优轮廓系数 S_year(k*) < 0.18，则判定年度季节性差异不显著，放弃分段，将所有周归为一类，标记为"常规期"。否则采纳 k* 分类结果。

对 k* 个簇按周平均负荷降序排列，赋予年度峰谷类别名称：周平均负荷最高的簇为"高负荷期"，周平均负荷最低的簇为"低负荷期"，中间簇依次命名为"中间负荷期 1""中间负荷期 2"等。全年每一天 d 从其所属周继承年度类别标签 c_season(d) ∈ {1(高负荷期), 2(中间负荷期1), ..., k*(低负荷期)}。

### 6.5 双维度组合典型日模型库构建

完成工休维度标注 c_workrest(d) 和年度维度标注 c_season(d) 后，对全年每一天进行双维度笛卡尔积组合，如图5所示。

每一天 d 获得一个二元组合模型编号：

```
model_id(d) = (c_workrest(d), c_season(d))               (17)
```

其物理含义为"该天所属的工休类型 + 该天所属的年度负荷水平段"的组合标签，例如 (1, 1) 表示"工作日-高负荷期"、(2, 3) 表示"休息日-低负荷期"。组合总数不超过 3 × 5 = 15 个，远小于 365，且每个组合具有清晰的语义。

对每个组合模型 (i, j)，设全年中属于该组合的全部天数为 N_{i,j}，其日负荷矩阵为 M_{i,j} ∈ R^{N_{i,j} × 24}。计算该组合模型的典型 24 小时负荷曲线为该组合下所有天的小时均值：

```
μ_{i,j}[h] = (1/N_{i,j}) × Σ_{d ∈ group(i,j)} M[d, h]    (18)
```

式中 h = 0, 1, ..., 23。μ_{i,j} 构成该组合模型的 24 点典型负荷曲线。

由此构建组合典型日模型库，包含所有实际存在的组合模型条目，每个条目由组合模型编号 (i,j)、组合模型名称（如"工作日-高负荷期"）、包含天数 N_{i,j} 和 24 点典型曲线 μ_{i,j} 组成。同时生成全年逐日模型映射表，记录每一天 d 的日期、星期、对应组合模型编号及名称，供下游求解器按日索引查询模型曲线。

### 6.6 面向储能优化的标准化 Runtime 输出

为支撑下游储能优化求解器的标准化消费，将模型库和逐日映射表输出为双文件 runtime 格式。全年逐日模型映射表 `runtime_year_model_map.csv` 包含 365 行，每行结构为：

```
date, day_index, internal_model_id, external_model_id, model_name
```

其中 internal_model_id 为连续整数编号（0, 1, 2, ...），按组合模型编号的字典序排列，便于求解器以数组索引方式实现 O(1) 查询；external_model_id 保留原始组合模型编号 "(i,j)"，保持可追溯性。

模型曲线库 `runtime_model_library.csv` 包含 N_model 行，每行结构为：

```
internal_model_id, external_model_id, model_name, h00, h01, ..., h23
```

两个文件共同构成储能优化求解器可直接消费的负荷数据接口。求解器在年度仿真中，对第 d 天，通过 day_index = d 查找 internal_model_id，再从模型库中以 internal_model_id 为索引获取 24 小时曲线，完成逐日负荷参数加载。

### 6.7 差异化双路径扩展：居民节点建模

对于配电网中与工商业用户用电行为驱动机制截然不同的居民节点，本发明还提供了差异化的建模路径作为扩展实施例，如图6所示。

居民用户用电行为受生活作息和季节气候驱动，个体差异大、无明显工休边界。因此居民路径采用多维特征聚合聚类策略，与工商业路径的层次化时序分解形成互补。

居民路径的特征工程从时间、形状、统计三个域提取共计 40 维特征：24 维形状特征（同工商业）、5 维统计特征（均值/峰值/谷值/标准差/峰谷差）、1 维日负荷率、5 维时段占比特征（夜间 0-6h / 早高峰 6-10h / 白天 10-17h / 晚高峰 17-22h / 深夜 22-24h 各时段负荷占日总负荷的比例）、2 维日期类型旗标（是否周末、是否节假日）、2 维月份循环编码（sin(2π·m/12)、cos(2π·m/12)，其中 m 为月份）。

对标准化后的 40 维特征执行自适应 KMeans 聚类，在 k = 2 至 8 范围内以轮廓系数择优。若最优轮廓系数低于阈值 0.08，则回退为 1 类模型。聚类完成后，对各模型自动生成三段式语义名称——季节属性（冬季/夏季/过渡季/全年，由模型内天数的月份分布确定）+ 日期属性（工作日型/周末型/节假日型/混合日型，由周末和节假日占比确定）+ 曲线形状属性（平稳型/早晚双峰型/晚高峰型/白天高平台型/夜间偏高型/综合波动型，由 24 点曲线的时段分布特征确定）——例如"夏季工作日型晚高峰型"。居民路径的最终输出同样转换为统一的 runtime 双文件格式，与工商业路径汇入同一储能优化求解管道。

### 6.8 算法详细程度说明

上述各模块中，KMeans 聚类均采用 scikit-learn 标准实现（n_init=10, random_state=42），标准化采用 StandardScaler（零均值单位方差）。轮廓系数计算公式为标准定义。本专利的技术贡献在于上述模块的组合逻辑、阈值设计、模式编码规则、三级修正策略和双维度交叉组合方法，而非 KMeans 或轮廓系数等通用算法本身。本领域技术人员根据上述公式、流程描述和参数设定，可在 Python 科学计算环境中完整复现本方法。

## 八、本专利的效果

（1）通过层次化工休模式识别方法（逐周聚类→模式串编码→多数投票→三级修正），有效解决了单周独立聚类缺乏全局视角导致的异常周误判问题。三重阈值联合判别从源头避免了对工休差异不显著周的错误二分类；以轮廓系数 0.15 为界区分"特殊周"与"可修正周"，使真实生产异常得以保留、聚类噪声得以纠正，最终工休标注准确率相比简单二分类方法有显著提升。

（2）通过双维度独立分解后组合交叉的模型库构建方法，避免了将工休维度与季节维度混为一维聚类导致的模型语义混杂问题。最终每个组合模型具有明确的"作息类型+负荷水平"语义标签（如"工作日-高负荷期"），使下游储能调度策略可追溯到具体行为类型，实现了"白箱"可解释的负荷-储能联合决策。

（3）本方法本质为"负荷结构建模"而非"负荷数值预测"——提取的是工休节奏和季节性负荷水平等级等跨年稳定的结构性规律，而非某具体年份的精确小时值。这些结构性规律源自企业的生产作息制度（制度不变则模式不变）和季节性气候特征（气候带不变则季节分段相对稳定），具有跨年可复用性。因此，基于本方法构建的典型年负荷模型驱动 10 至 15 年储能全生命周期仿真时，不会将单一年份的偶然极端事件（限电、停产、设备故障等）外推至整个项目周期，NPV 和回收期等财务指标对不同年景的输入数据具有更强的鲁棒性。

（4）差异化双路径框架分别适配工商业和居民两类节点的用电行为驱动机制——工商业路径以层次化工休识别为核心，居民路径以多维特征聚合聚类为核心——使配电网中不同性质的节点均能以最适合其行为特征的方式构建负荷模型，两条路径在完整性和精确性之间取得了平衡。

（5）统一的 runtime 双文件标准化输出格式，使负荷建模结果与下游储能优化求解器实现了接口解耦——任何遵循该格式的建模方法均可接入求解器，求解器仅依赖 internal_model_id 这一整数索引实现 O(1) 的模型查询，兼顾了接口简洁性和计算效率。
