# 创新点清单：工商业配储优化平台

## 项目概述

本项目是一个面向工商业用户的配电网电池储能系统优化配置平台，核心求解引擎采用多目标遗传算法（GA）对储能功率/容量进行优化，结合日前凸优化调度、滚动实时执行、OpenDSS 配电网潮流校核和全生命周期财务评价，实现储能系统在经济性与配电网安全性之间的最优权衡。

---

## 候选创新点及调研验证

### 创新点 1：GA-OpenDSS 双阶段协同评估框架

**技术问题**：储能优化配置需要遗传算法进行数千次候选方案评估，但每次评估若包含完整的 365 天 OpenDSS 配电网潮流仿真则计算量过大（单次全年 OpenDSS 仿真需数分钟），使 GA + 精确潮流校核的组合在计算上不可行。

**本项目的技术方案**：

提出"快评代理（fast_proxy）+ 全年重校核（full_recheck）"双阶段评估策略。第一阶段采用自适应代表日聚类——基于负荷均值、负荷标准差和电价均值三个维度对 365 天进行签名分组，每组仅仿真一个代表日，将全年仿真压缩至数十个代表日；该阶段即可调用 OpenDSS 进行小时级网络约束校验。第二阶段仅对通过经济性门槛（回收期 ≤ 阈值、NPV/投资比 ≥ 阈值、年度现金流转正）的快评候选方案触发完整的 365 天 + OpenDSS 重校核。两阶段之间通过自动触发机制无缝衔接，GA 优化全程均可获得 OpenDSS 网络约束反馈，但仅在必要时付出完整计算代价。求解结束后对 Top-K Pareto 候选执行 OpenDSS 全年重校核并完成闭环收敛校验。

**相关现有技术调研**：

- 专利 CN119171530A（国网福建，2024）："配电网多目标联合规划方法"采用改进 NSGA-III + 逻辑混沌映射 + 置信区间上界局部搜索进行光伏+储能联合规划，但其潮流计算采用常规牛顿-拉夫逊法嵌套，未涉及双阶段评估策略或 OpenDSS 协同仿真。
- 专利 CN202210862513.8（国网湖北，2022）："基于改进遗传算法的电力系统多目标储能优化方法"在源-网-荷三侧配置储能并进行多目标优化，但未涉及配电网潮流校核的 GA 协同框架。
- 分布式电源优化配置方法专利（2024）：采用遗传算法嵌套牛顿-拉夫逊法进行潮流计算，未采用双阶段策略或 OpenDSS。
- Teimourzadeh 等，学术论文：OpenDSS + GA 的储能优化在学术文献中存在，但均为单阶段（全部候选均做 OpenDSS 仿真），未采用快评代理 → 重校核的分层策略。

**新颖性分析**：

现有专利和学术文献中：
- GA + OpenDSS 的组合存在于学术论文中，但均为"每个候选方案都完整仿真"的单阶段模式；
- 双阶段评估（轻量代理 → 精确校核）在 GA 优化领域是通用思路，但在"储能 GA 优化 + OpenDSS 配电网潮流校核"这一特定组合中的应用未见现有专利或文献；
- 本方案的差异化在于：(a) 代表日聚类使用负荷/电价多维度签名而非单一 k-means；(b) 快评阶段即引入 OpenDSS 校核（非简化的线性约束）；(c) 重校核触发条件基于经济指标而非简单的可行性判断；(d) 求解后闭环收敛校验机制。

本方案具有可论证的新颖性。需注意在撰写时强调"GA + OpenDSS"组合中双阶段分层评估策略的独特性，而非 GA 或 OpenDSS 各自本身。

**项目中的体现位置**：

- `storage_engine_project/simulation/annual_operation_kernel.py:721-758`：`_build_representative_day_groups()` 自适应代表日聚类
- `storage_engine_project/main.py:484-563`：`_ensure_topk_full_recheck()` Top-K 重校核闭环

### 创新点 2：多维归一化 Pareto 折中解选择与可调安全-经济权衡机制

**技术问题**：多目标储能优化生成 Pareto 前沿后，如何从众多非支配解中选出单一最终方案是一个经典难题。现有方法（TOPSIS、模糊隶属度、加权求和）通常采用固定权重或需要决策者直接指定各目标权重，缺乏直观且可连续调节的单一控制参数。

**本项目的技术方案**：

提出基于四项归一化指标（Pareto 前沿距离、回收期、安全性、NPV）的加权评分法，通过单一连续参数 s ∈ [0,1]（安全-经济权衡系数）动态调节四项权重：

- 距离权重 = 0.45 - 0.15s
- 回收期权重 = 0.35 - 0.30s
- 安全性权重 = 0.00 + 0.50s
- NPV 权重 = 0.20 - 0.15s

当 s=0（纯经济模式）时，安全性权重为 0，距离和回收期权重最高；当 s=1（纯安全模式）时,安全性权重升至 0.50，回收期权重降至 0.05。安全指标综合了变压器越限小时数、越限功率、电压越限小时数、线路过载小时数和等效循环次数，并扣减储能接入后安全改善量（delta_safety），使得客观衡量"储能对配电网安全性是改善还是恶化"成为可能。

前端通过 SolverPage 上的滑块组件将此参数暴露给用户，实现直观的交互式方案选择。

**相关现有技术调研**：

- 专利"一种储能系统选址定容方法"（jigao616.com）：采用双目标（停电损失减少量 ΔKₗ 为安全性指标、经济效益 Eco 为经济性指标）Pareto 优化，使用模糊隶属度函数计算综合满意度进行方案选择。为双目标、模糊隶属度方法，非四目标、线性权重调节。
- 专利 CN121581287A（华能深圳，2026）：采用 NSGA 生成 Pareto 解集后使用 TOPSIS 法进行综合评分排序，克服主观赋权弊端，但 TOPSIS 为固定权重方法，不支持用户通过单一参数连续调节偏好。
- Zhang 等学术论文：Pareto 前沿通常通过加权求和或 ε-约束法转换为单目标，或通过模糊决策选取折中解，少有多维权重的单一参数连续调节机制。

**新颖性分析**：

- 现有储能 Pareto 优化专利中，折中解选择普遍采用 TOPSIS、模糊隶属度等固定或半固定权重方法，未见通过单一连续参数 s 动态调节多维权重的方案；
- 本方案的安全指标设计（综合变压器/电压/线路/循环 + 扣减安全改善量）比单纯的电压偏差或网损指标更全面；
- 通过前端滑块实时调节安全-经济偏好的交互模式在储能优化软件产品中具有差异化特征；
- 需注意：Pareto 前沿 + 加权选优的总体框架是公知技术，创新点在于四维归一化 + 单参数连续调节 + 安全改善增量指标的组合设计。

**项目中的体现位置**：
- `storage_engine_project/optimization/pareto_utils.py:48-106`：`select_best_compromise()` 四维加权评分逻辑
- `storage_engine_project/optimization/storage_fitness_evaluator.py:412-446`：`_build_objective_vector()` 安全代理指标计算
- `frontend/src/pages/workspace/SolverPage.tsx`：安全-经济滑块组件

### 创新点 3：基于多维签名的自适应代表日聚类与递推 SOC 的年度快评方法

**技术问题**：全年 8760 小时逐时调度仿真计算量大，在 GA 迭代中不可行。现有典型日/代表日方法通常采用 k-means 聚类（按负荷曲线形状聚类），但存在两个不足：(1) 未能同时考虑负荷水平和电价水平对储能套利决策的影响；(2) 代表日之间的 SOC 递推（carry-over）处理不当会导致储能长期调度行为的失真。

**本项目的技术方案**：

提出基于三维签名（负荷均值 / 50、负荷标准差 / 30、电价均值 × 20）的自适应分组方法，将 365 天映射到离散签名空间，相同签名的天归为一组，每组仅仿真签名空间中的首日作为代表日。该方法具有以下特点：

1. **多维经济-负荷联合签名**：同时编码负荷的量级（均值）、波动性（标准差）和电价水平（均值），使分组结果在储能套利决策空间（而非仅负荷曲线形状空间）中具有同质性；
2. **组内递推 SOC**：快评模式下通过 `fast_proxy_carry_soc` 参数控制在代表日之间是否递推 SOC 状态，保持储能长期运行的 SOC 连续性；
3. **非参数化分组**：不使用 k-means 等需要预设聚类数的方法，分组粒度由签名空间的离散化步长自然决定（50kW、30kW、0.05 元/kWh），自适应于数据特征。

**相关现有技术调研**：

- ScienceDirect 论文（2024）："Representative days and hours with piecewise linear transitions for power system planning"——使用分层聚类选取代表日并采用分段线性调度，但聚焦于发电规划而非储能优化。
- MIT 博士论文（Barbar，2022）：使用 autoencoder + k-means 进行多变量时间序列降维后聚类，并讨论长时储能的跨日 SOC 递推问题。
- 专利 CN202411319624（山东大学）："保留多维时序特征的场景分解聚类与评估方法"——采用季节分解 + SOM 神经网络 + k-means 进行场景缩减，用于氢储能+电池储能联合规划。
- 专利"一种面向储能规划的典型复合型场景缩减方法"：分别对风电、光伏、负荷做 k-means 聚类后重新组合为复合典型场景。

**新颖性分析**：

- 现有代表日聚类方法以 k-means 及其变体为主流，本方案采用固定步长签名空间映射的方法，具有非参数化、结果可解释性强的特点；
- 三维签名中显式包含电价维度在现有专利/文献中较为少见——现有方法多聚焦于负荷/新能源曲线的形状特征，未将电价信号纳入聚类判据；
- 然而，"代表日缩减 + 年度仿真"的总体框架在学术界已是标准做法（即 computational reduction for annual dispatch），因此本创新点的强度在于签名设计的具体方法及其与双阶段评估框架的集成方式,而非代表日聚类的概念本身。

**项目中的体现位置**：
- `storage_engine_project/simulation/annual_operation_kernel.py:721-758`：`_build_representative_day_groups()` 签名构建与分组逻辑
- `storage_engine_project/simulation/annual_operation_kernel.py:410-447`：快评模式代表日遍历与 SOC 递推

### 创新点 4：OpenDSS COM 增量编辑优化与自动回退的全天逐时网络约束校验

**技术问题**：通过 COM 接口调用 OpenDSS 进行配电网潮流校核时，若每个小时都重新编译整个 Master.dss 电路文件，编译开销将随仿真小时数线性增长（365 天 × 24 小时 = 8760 次编译）。如何在保证网络约束评估精度的前提下降低 COM 调用开销？

**本项目的技术方案**：

提出基于"日首小时全量编译 + 同日内后续小时 Edit 增量更新 + 收敛失败自动回退重编译"的三层策略：

1. **日级编译缓存**：每天的第一个小时执行 `Compile [Master.dss]` 完整编译，后续 23 个小时仅通过 `Edit Load.xxx kW=...` / `Edit Storage.__GPT_STORAGE kW=...` 命令增量更新负荷和储能元件的功率值；
2. **临时元件状态管理**：对于反向潮流（PV 过剩导致负荷节点净注入）场景，通过创建临时 Generator 元件（`New Generator.__GPT_TMP_BG_GEN_xxx`）模拟功率注入，每小时结束后通过 `Edit Generator.xxx kW=0` 清零而非 Delete（避免某些 OpenDSS COM 版本中 Delete 命令触发模态警告）；
3. **收敛失败自动回退**：若 Edit 后的潮流求解不收敛，自动回退到完整 Compile + 全部负荷重设的路径，并记录回退次数用于诊断。

**相关现有技术调研**：

- 在专利数据库中未检索到明确涉及"OpenDSS COM 增量编辑优化"的专利文献。
- 学术文献中，OpenDSS 的 COM 接口主要用于离线分析或单次批量潮流计算，增量 Edit 策略通常被视为工程实现细节而未被作为方法创新点提出。
- OpenDSS 官方文档提到 `Edit` 命令可用于参数修改，但未提供逐时增量更新的优化策略。

**新颖性分析**：

- 本方案属于工程实现层面的优化创新，解决了 COM 接口在实际大规模仿真中的性能瓶颈；
- "增量更新 + 自动回退"的策略模式本身是通用的工程实践，但将其系统化地应用于 OpenDSS COM 接口的逐时配电网约束校验场景，并配合日级编译缓存和临时元件状态管理，在现有专利/文献中未见相同方案；
- 建议作为前述创新点 1（双阶段评估框架）中的关键技术细节出现，而非独立的专利创新点。

**项目中的体现位置**：
- `storage_engine_project/simulation/opendss_network_constraint_oracle.py:708-786`：`get_hour_constraint()` 中的日首小时判断与增量更新逻辑
- `storage_engine_project/simulation/opendss_network_constraint_oracle.py:151-169`：`edit_storage_dispatch()` 增量储能更新
- `storage_engine_project/simulation/opendss_network_constraint_oracle.py:171-178`：`disable_temp_generators()` 临时元件清零策略

## 建议

综合以上分析，**创新点 1（GA-OpenDSS 双阶段协同评估框架）** 具有最强的新颖性和最完整的专利保护价值，建议作为专利撰写的首选目标创新点。创新点 2（多维归一化 Pareto 折中解选择）可作为创新点 1 的补充或作为从属创新点。

创新点 3 和 4 的技术贡献相对较小，建议作为创新点 1 技术方案中的方法细节出现，而非独立申请。
