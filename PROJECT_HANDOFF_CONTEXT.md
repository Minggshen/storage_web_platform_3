## 1. 项目定位

本项目是 `storage_web_platform_2` 的备份和继续优化版本，命名为 `storage_web_platform_3`。

核心目标：面向某一个工商业用户进行单独配储优化。用户在拓扑建模/计算运行流程中人为指定一个候选配储目标负荷，其他负荷作为背景负荷参与 OpenDSS 全网潮流计算。

最终面向规划/运维人员输出四类交付物：

1. 储能配置方案：装多大、装在哪、几套。
2. 年运行情况：8760h 充放电曲线、SOC 轨迹、等效循环、利用率。
3. 经济性分析：NPV、IRR、回收期、LCOS、年度收益/成本拆分。
4. 配电网承载力/供电能力变化：储前/储后电压越限、线路/配变过载、网损变化、风险元件。

## 4. 关键业务约束和设计决定

1. 储能对象建模
   - 用户不需要在拓扑图中手动添加储能元件。
   - 用户只需要选择候选配储目标负荷。
   - 后端/求解器会在 OpenDSS 仿真时为该目标节点生成明确的 Storage 元件。

2. 单目标用户配储流程
   - 计算运行页的候选目标下拉框来自拓扑建模中设置为候选配储目标的负荷。
   - `target_id` 指定本次优化的目标负荷。
   - 其他负荷不是被忽略，而是作为背景负荷参与全网潮流。

3. 容量边界
   - 已删除用户手动覆盖字段：
     - `storage_power_min_kw`
     - `storage_power_max_kw`
     - `storage_energy_min_kwh`
     - `storage_energy_max_kwh`
   - 搜索空间边界由 `backend/services/search_space_inference_service.py` 自动推断。
   - LOAD 面板中展示的是只读推断结果，不再允许用户手动改边界。

4. OpenDSS 模型
   - 已朝三相平衡模型改造。
   - 已加入上级电网短路容量与 X/R 参数。
   - 主变参数已完善，先固定 tap，后续可考虑 RegControl。
   - 线路型号库和三相 LineCode 已重做。
   - 储能仿真已从临时 Load/Generator 逐步改为原生 Storage。
   - OpenDSS 实编译探测结果写入 `dss_compile_summary.json`，并在构建校验页展示。

5. 经济性参数
   - 经济参数应作为项目全局参数设置，不应散落到每个负荷节点。
   - 前端全局经济参数保存后，构建时写入候选配储目标的 registry 行。
   - 收益/成本口径已包含：
     - 套利收益
     - 需量收益
     - 辅助服务收益
     - 容量收益
     - 降损收益
     - 运维成本
     - 退化成本
     - 更换成本
     - 政府补贴

检查内容包括：

- SOC 是否越界。
- SOC/能量守恒异常。
- OpenDSS 是否收敛。
- 储前/储后电压、线路、主变越限。

### 5.5 配电网影响分析

已具备储前/储后对比基础：

- 节点电压越限。
- 线路过载。
- 主变过载。
- 储能接入后改善/恶化归因。
- Top 风险元件。
- 目标接入点局部影响。

关键口径：储能方案可行性不能只看全网总风险，而应重点看储能接入节点、目标用户配变、接入线路、相关馈线是否因储能导致新增越限或过载。

### 5.7 最近一次超时修复

问题：

用户在结果展示页选择任务 `fbddc1ac9d14` 时，接口超时：

```text
请求超时：http://127.0.0.1:8001/api/solver/project/f53eb4ccd9cf/summary?task_id=fbddc1ac9d14 在 60 秒内没有响应
```

根因：

该任务 OpenDSS 逐时 trace 文件很大：

- `best_bus_voltage_trace.csv` 约 97 MB。
- `best_line_loading_trace.csv` 约 104 MB。

后端原先在生成结果页图表时把大 CSV 整表读入内存，导致切换历史任务时阻塞。

已修改文件：

```text
D:\storage_web_platform_3\backend\services\solver_execution_service.py
```

已做修改：

- 结果页任务切换只读任务元数据，不再读取 stdout 做进度解析。
- 大 CSV 改成流式逐行汇总。
- CSV 文件预览只读取前 50 行，大文件不再整表扫描。

验证：

- 已通过 Python 内存语法编译检查。
- 由于当前后端进程锁住 `__pycache__`，`py_compile` 写入缓存时可能报权限错误，不代表语法失败。

后续验证：

- 重启后端后，重新选择任务 `fbddc1ac9d14` 测试是否仍超时。
- 如果仍慢，下一步应把拓扑图汇总结果缓存为 JSON，首次生成后复用缓存。
